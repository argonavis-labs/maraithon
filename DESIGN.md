# Design

Maraithon is an operational app. Interfaces should be quiet, compact, and built for repeated use.

## Product UI

- Prefer rows, tables, and concise panels over decorative cards.
- Summary pages should show only the highest-signal rollup. Connector summary rows show the connector, status, and connected account count.
- Put account-level connector details on the connector detail page as separate rows with status, last update, and actions.
- Make drill-in rows clickable. Keep secondary actions small and aligned to the right.
- Avoid gradient hero sections, marketing-style copy, nested cards, oversized headings, and heavy shadows.
- Use restrained borders, consistent spacing, and 8px or smaller border radii.
- Keep labels direct. If something requires action, say that clearly and place the suggested action nearby.
